## 6. Next required phase

The next implementation phase should be:

```text
V0.1a completion: exec-only runner alignment
```

Primary goal:

- connect CLI `run` to a true one-shot acpx exec runner under role-bound authorization, with fake-subprocess tests first and no persistent sessions.

Acceptance:

- all existing gates pass;
- fake subprocess tests cover success/fail/timeout/watchdog/stderr/stdout;
- dry-run remains available if desired;
- real `run` path no longer refuses solely because real launch is disabled, once the phase explicitly approves one-shot local exec-only launch;
- no Sachima/Gateway/IM/live integration is added.
