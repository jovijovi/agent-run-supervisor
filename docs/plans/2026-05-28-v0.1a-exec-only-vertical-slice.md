# agent-run-supervisor V0.1a Exec-Only Vertical Slice Implementation Plan

> **For Hermes:** Claude Code is the main worker for implementation/debugging. Codex CLI is the primary reviewer. Hermes controls scope, verification, and arbitration.

**Goal:** Implement the first exec-only Python library + dev CLI vertical slice on top of Phase -1 `acpx@0.10.0` contract fixtures.

**Architecture:** Keep the package small and role-centric. The library validates `AgentRoleSpec`, compiles role permissions to acpx argv/policy, classifies exits, parses the Phase -1 observed JSON-RPC stdout, writes redacted run artifacts with secure permissions, and exposes `validate-role`, `replay`, `doctor`, and `run` CLI commands. Business PASS/BLOCK parsing remains out of scope.

**Tech Stack:** Python 3.11+, stdlib only for runtime, pytest for tests.

## Approved scope

- Package implementation under `src/agent_run_supervisor/`.
- Tests under `tests/`.
- Docs/dev-log updates for V0.1a evidence.
- CLI commands:
  - `agent-run-supervisor validate-role <role-file>`
  - `agent-run-supervisor replay <events.ndjson>`
  - `agent-run-supervisor doctor [--role <role-file>] [--fixtures <dir>]`
  - `agent-run-supervisor run --role <role-file> --prompt-file <file> [--cwd <dir>] [--no-real-run]`
- `run --no-real-run` must compile and persist a safe dry-run result without launching acpx.
- Real `run` may support subprocess execution, but tests must use fake subprocesses/fixtures by default.

## Explicit non-approvals

- No persistent sessions.
- No session registry/locking/stale-lock recovery.
- No Sachima behavior integration.
- No real AGENT automatic replies.
- No public ingress, real delivery, Gateway lifecycle operations, production config writes, or live/default-on behavior.
- No `@all`, participant management UI, or agent-to-agent routing.
- Do not claim `allowed_roots` is an OS/filesystem sandbox.

## Acceptance criteria

1. `AgentRoleSpec` validates the V0.1a schema with strict defaults and clear errors.
2. Policy compiler emits acpx args with `--format json --json-strict --suppress-reads --timeout <n> --max-turns <n> --permission-policy <json> --non-interactive-permissions fail` plus `--no-terminal` when role disallows terminal.
3. `allowed_roots` only validates cwd/config; docs/result metadata state it is not a security boundary.
4. `AcpxExitClassifier` table-tests exit `0/1/2/3/4/5/130/unknown`, with nonzero never completed.
5. Parser replays Phase -1 observed stdout fixtures and:
   - assembles `final_message` from `agent_message_chunk` text;
   - records usage updates;
   - maps permission request/denial updates;
   - fails malformed NDJSON as `protocol_error`;
   - records unknown update key/type summaries only;
   - rejects management `status=no-session` JSON as exec success;
   - keeps `business_verdict=null`.
6. EventStore writes `.agent-run-supervisor/runs/<run_id>/` with directory mode `0700` and artifact mode `0600`.
7. Redaction covers prompt, stderr, env, argv, metadata, events, and result; no secret-shaped values in committed tests/fixtures beyond split synthetic strings.
8. CLI commands have tests and return nonzero on invalid inputs.
9. `doctor` checks Python/Node/acpx version shape, fixture replay, EventStore permission probe, and role validation where provided. It must not launch real agents.
10. Fresh gates pass: `python3 -m pytest -q`, `python3 -m compileall -q src scripts tests`, fixture validator, secret scan, `git diff --check`.
