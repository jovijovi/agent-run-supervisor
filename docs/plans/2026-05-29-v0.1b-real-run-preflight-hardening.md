# agent-run-supervisor V0.1b Real-Run Preflight Hardening Plan

> **For Hermes:** Claude Code is the main implementation/debugging worker. Codex CLI is the primary reviewer. Hermes controls scope, verification, evidence, and arbitration.

**Goal:** Harden the preflight gates that must hold before any future real-run phase: actual Node/acpx probing in `doctor`, cwd-vs-allowed-roots fail-closed validation, and explicit real-run refusal unless a future phase grants a launch gate.

**Architecture:** Keep the V0.1a exec-only shape. Add small stdlib-only helpers for environment probing and workspace cwd validation, integrate them into `doctor` and `run`, and keep all real-launch behavior disabled. `allowed_roots` remains cwd/config validation only, never an OS sandbox claim.

**Tech Stack:** Python 3.11+, stdlib runtime, pytest for tests.

## Current project position

- Phase -1: complete with `acpx@0.10.0` fixtures and validator.
- V0.1a: complete on `main` with exec-only vertical slice and AI_FLOW governance.
- Current allowed next request in `docs/roadmap/current-status.md`: V0.1b may harden real-run preflight, actual Node/acpx probing, cwd-vs-allowed-roots validation, or HITL/manual execution design gate.
- User approval for this phase: implement V0.1b real-run preflight hardening only.

## Approved scope

- Actual `doctor` probes for:
  - Python version already reported;
  - `node --version` detected version and minimum requirement check;
  - `acpx --version` detected version and exact expected `0.10.0` check when available.
- Cwd-vs-allowed-roots validation:
  - default cwd and CLI `--cwd` override must resolve inside one configured allowed root;
  - validation fails closed with a safe error payload/message when outside allowed roots;
  - validation must not describe allowed roots as an OS sandbox.
- Real-run gate:
  - `run` without `--no-real-run` still refuses before launching anything;
  - refusal happens before artifact creation or subprocess launch;
  - output is deterministic enough for tests and future HITL design.
- Docs updates:
  - plan and dev log;
  - `GOAL.md` / `docs/roadmap/current-status.md` after implementation;
  - generated docs index and drift report.

## Explicit non-approvals

This phase does **not** approve:

- real AGENT execution;
- subprocess launch of acpx for `run`;
- persistent sessions;
- session registry, locking, stale-lock recovery, or multi-turn context retention;
- Sachima integration;
- real AGENT automatic replies;
- public ingress or real IM delivery;
- Gateway restart/reload/replace;
- production config writes;
- live/default-on behavior;
- worker auto-routing;
- participant persistence or management UI;
- `@all` fanout;
- agent-to-agent automatic routing;
- trusted Markdown/HTML rendering;
- treating `allowed_roots` as an OS/filesystem sandbox.

## Acceptance criteria

1. `doctor` emits structured `node_probe` and `acpx_probe` objects including `binary`, `available`, detected `version`, expected/minimum version fields, `ok`, and sanitized error detail if unavailable.
2. `doctor` does not launch a real agent and keeps `launched_real_agent=false`.
3. `doctor` exit status fails only for fixture/permission/role failures, or for hard preflight checks that this phase decides should block; missing local `acpx` may be represented as unavailable without launching agents, because CI may rely on `npx` rather than global `acpx`.
4. `run --no-real-run` succeeds when effective cwd is inside `allowed_roots` and persists cwd validation metadata/disclaimer.
5. `run --no-real-run` fails closed when effective cwd is outside `allowed_roots`, without creating run artifacts.
6. `run` without `--no-real-run` returns a stable refusal code/message and never creates run artifacts or launches acpx.
7. Tests prove RED/GREEN for doctor probes, cwd gate, and real-run refusal.
8. Fresh gates pass:
   - `python3 scripts/validate_contract_fixtures.py fixtures/acpx-0.10.0`
   - `python3 -m pytest -q`
   - `python3 -m compileall -q src scripts tests`
   - `PYTHONPATH=src python3 -m agent_run_supervisor doctor`
   - `PYTHONPATH=src python3 -m agent_run_supervisor replay fixtures/acpx-0.10.0/success-codex-sentinel/stdout.ndjson`
   - `python tools/build_docs_index.py --check`
   - `python tools/docs_drift_signal.py --check`
   - `git diff --check`
   - changed-line secret scan = 0
   - changed-line static dangerous-pattern scan = 0
9. Codex primary review and Claude Code auxiliary review report no blockers.

## Risks and controls

- **Risk:** local `acpx` may not be installed globally. **Control:** `doctor` should report availability and exact version when available without requiring global install unless a role explicitly points at a binary. Future real-run can decide stricter policy.
- **Risk:** cwd validation is confused with sandboxing. **Control:** metadata/docs/test assertions keep `allowed_roots_security_boundary=false` and include the disclaimer.
- **Risk:** real-run refusal accidentally creates artifacts. **Control:** tests assert no run directory is created when `--no-real-run` is missing.
- **Risk:** a helper uses subprocess for launch. **Control:** only version probes run `--version`; no acpx `exec` or runner launch is introduced.
